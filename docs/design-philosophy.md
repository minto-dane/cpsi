# 設計思想

CPSI は人間が理解しやすい軽量なパッケージマネージャーを目指します。

## 重視すること

1. パッケージは自己完結である
2. ビルドと配布を分離する
3. シンプルな依存解決
4. 軽量な署名機構
5. 最小限のスクリプト実行
6. Parquet による統一的データ管理
7. ロールバックを持たない軽量設計
8. 人間が理解しやすい仕様

## 非目標

現時点では、以下を積極的には扱いません。

- プレリリース版の複雑なバージョン解決
- `^`、`~`、`*` などの依存バージョン記法
- ロールバック付きトランザクション
- リポジトリメタデータ署名
- conflicts の解決

これらは実装の単純さ、保守性、動作の分かりやすさを優先するためです。
