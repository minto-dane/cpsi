# パッケージスクリプト

CPSI のスクリプトシステムは `post` を主役とします。

## `post`

`post` はインストール後に実行します。

用途例:

- icon cache 更新
- MIME database 更新
- font cache 更新

## `pre`

`pre` は必要な場合のみ実装します。

存在しない場合、CPSI は `pre` を無視します。

## `post` 失敗時の扱い

流れ:

```text
展開成功
↓
post 実行
↓
失敗
↓
警告表示
↓
終了
```

結果:

- パッケージはインストール済み扱いにする
- ロールバックしない

理由:

- シンプルな実装を維持するため
- `post` は補助処理であるため
