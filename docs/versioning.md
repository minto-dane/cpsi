# バージョンとアーキテクチャ

## release

`release` は `k` を付けてファイル名に含めます。

例:

```text
1.0-k1
1.0-k2
1.0-k3
```

`release` は以下の用途で増加します。

- パッチ更新
- ビルド設定変更
- 依存関係修正
- パッケージング修正

上流ソースコードが同一でも増加できます。

## バージョン比較

比較順序:

```text
version
↓
release
```

例:

```text
1.0-k1 < 1.0-k2
1.0-k2 < 1.1-k1
```

## プレリリース

現時点では以下のようなプレリリース表記は扱いません。

```text
1.0-alpha
1.0-beta
1.0-rc1
```

理由:

- 依存解決の複雑化を避けるため
- CPS Linux のリポジトリでは正式版のみを配布するため

## アーキテクチャ識別子

現時点で想定するアーキテクチャ:

```text
x86_64
aarch64
riscv64
```

アーキテクチャはパッケージ名と metadata の両方に含めます。

理由:

- リポジトリ管理の簡略化
- 人間が見て識別しやすい
